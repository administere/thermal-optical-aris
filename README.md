# 热光混合处理器 — 工程验证

> 用光做计算，用热来控制光。一份诚实的工程分析。

## 这是什么

我在做一个新型光计算芯片的可行性分析。基本想法很简单：

```
一束激光 → 穿过一片特殊的薄膜 → 被探测器接收
              ↑
         薄膜被加热到 242°C
         加热改变了薄膜对光的"折射能力"
         于是光的强弱被精确控制
         这就是"热光计算"
```

关键技巧是**光子复用**：一个光脉冲能穿过 2048 个调制点，相当于做了 2048 次乘法。光在这里既传递信息，又维持温度。

## 六个核心问题

| # | 问题 | 目前结论 | 
|---|------|:--:|
| 1 | 242°C 能维持住吗？ | 实验证实材料可以，但靠 VCSEL 自身加热不够，需要外加热源 |
| 2 | 计算速度够快吗？ | 普通热光方式约 0.5 次/秒写权重，跟 H100 差不多 |
| 3 | 光信号够强吗？ | 够，SNR 32 dB |
| 4 | 电路复杂吗？ | 不复杂，2048 个 ADC 就够了 |
| 5 | 省电吗？ | 如果用量子拍频机制（17.6 GHz），能省 ~200 倍；普通方式只省几倍 |
| 6 | 能造出来吗？ | 需要先做实验验证，第一步约 $5K，2 周 |

## 材料：DiSubPc·C70

四川大学 2026 年在《Nature Photonics》上发了一篇论文，发现了一种叫 DiSubPc·C70 的有机共晶材料。它被光照后能自己升温到 242°C，而且内部电子会产生每秒 176 亿次的"量子拍频"。

论文只用它来做光热转换（烧水、淡化海水）。我第一个想到用它来做光计算——这个材料在 242°C 下对光的折射能力会随温度变化，可以用来做光学乘法。

### 这个材料有什么特别的

- **17.6 GHz 量子拍频**：电子在两种状态之间每秒振荡 176 亿次。如果能把这种振荡转化成光信号的强弱变化，计算速度会比普通方式快 400 倍。
- **极性晶体结构**：材料的分子排列不对称，这是产生量子拍频的结构基础。跟它类似的 C60 版本是对称的，拍频只有 6 GHz。

### 这个材料有什么问题

- **850nm 激光吸收太弱**：材料最适合的吸收波长是 570nm 左右（绿光），850nm 的近红外光吸收只有绿光的 1/230。我们用了数学方法估算 850nm 的吸收率，但不确定度很大。
- **242°C 下的性能没人测过**：所有光谱数据只测到室温（27°C），242°C 时材料会发生什么，全靠推测。
- **量子拍频还没被用来做计算**：论文只证明了拍频产生了热，没证明拍频能调制光信号。

## 替代材料 TiO₂

如果不追求量子拍频，只做"加热改变折射率"这件事，TiO₂（二氧化钛）是更好的选择：

- 折射率随温度变化的幅度是 DiSubPc·C70 的 3 倍
- 完全透明，光不会被浪费
- 熔点 1840°C，242°C 对它来说不算什么
- 标准半导体工艺就能做

但它没有任何量子效应。如果你只做普通热光计算，TiO₂ 更好。如果你想赌量子突破，DiSubPc·C70 是唯一的选项。

## 能耗对比

| 方式 | 每 dot product 能耗 | vs H100 |
|------|:--:|:--:|
| DiSubPc·C70 自加热 @ 850nm | 377,000 fJ | 8× |
| DiSubPc·C70 外加热 @ 570nm | 706,000 fJ | 4× |
| TiO₂ 外加热 @ 570nm | 693,000 fJ | 4× |
| **DiSubPc·C70 量子拍频** | **14,000 fJ** | **199×** |

> H100 GPU 做同样的事需要约 2,900,000 fJ。

**核心结论：普通热光方式只比 GPU 省几倍的电。只有量子拍频机制能实现数量级的突破。材料选 TiO₂ 还是 DiSubPc，差距只有 20%。但用不用量子拍频，差距是 50 倍。**

## 实验验证的优先顺序

1. **最关键**：在 242°C 下验证量子拍频能不能用来调制光信号
2. **次关键**：测量 TiO₂ 和 DiSubPc·C70 在 242°C 的实际折射率变化
3. **可以以后做**：精确测量 DiSubPc·C70 在 850nm 的吸收率
